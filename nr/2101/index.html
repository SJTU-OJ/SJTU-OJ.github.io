    2101 情书之谜

    题目

    Description

    大脸一直想看饼意写的情书。一次偶然的机会，大脸获得了情书的电子稿，但是电子稿被加密了，如下：

    经过大脸的研究，他发现这个图象表达的意思是，有一个长度为6的序列（From之后所示的序列），一开始有个光标处于第一个数字之下，每次我可以通过下方的六个按键使得原序列或光标变化，他发现六个按键(+, -, <, >, 1, 6)的作用分别如下： 

    +：  将光标所在位置上的数字加一；
-：  将光标所在位置上的数字减一；
<：  将光标左移一位，如果光标已经处于序列的最左端则无效；
>：  将光标右移一位，如果光标已经处于最右边则无效；
1：  将当前光标所在位置上的数字和序列第一位上的数字交换；
6：  将当前光标所在位置上的数字和序列最后一位上的数字交换；

    “最后的答案是从From之后的序列到To之后的序列的最小操作数！”大脸感到非常激动，然后昏迷不醒，临昏迷之前他把谜题告诉了你，希望你能够帮他解决该谜题的最后一步。

    Input Format

    仅有一行，包括两个长度为6的数字序列，第一个表示的是From之后的序列，第二个表示的是To之后的序列。

    Output Format

    仅有一个正整数，为最少需要的操作数。需要注意的是，最后光标的位置允许在任何一个位置。

    Sample Input 

    123456 654321

    Sample Output 

    11

    Sample Explanation 

    操作序列为6>1->+>->+1

    Limits

    Time limit: 2000ms, memory limit: 50000kb.
